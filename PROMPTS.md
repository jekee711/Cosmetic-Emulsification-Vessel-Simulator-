# 개발 로그 & 프롬프트 기록 (PROMPTS.md)

**프로젝트:** 화장품 O/W·W/O 에멀젼 호모믹서 스케일업 시뮬레이터 (V14)
**개발 방식:** 생성형 AI(LLM)를 활용한 반복적 개발 (V1 → V14)
**기술 스택:** Three.js 0.160.0 · 순수 HTML/CSS/JavaScript · 빌드 도구 없음

본 문서는 시뮬레이터를 V1부터 V14까지 개발하며 단계별로 무엇을 요청하고, 어떤 한계를
어떻게 극복했는지를 정리한 개발 로그다. 물리 모델은 아래 두 논문을 1차 근거로 삼았다.
각 단계의 핵심 요청 의도를 짧은 프롬프트 요지로 함께 기록한다.

## 0. 참고 논문 및 적용 매핑

**[논문 A] Calvo et al. (2024)**, *A population balance model for cosmetic emulsion design*, Chem. Eng. Sci. 287:119737
대상이 O/W 화장품 에멀젼이라 도메인이 정확히 일치. 채택 모델: 에너지 소산율 ε=b₁N³L²(식 7),
파괴 빈도 g=관성항+점성항(K₁~K₄), 이항 파괴 딸방울 분포, Fixed-Pivot 이산화(65 노드, 0.01~40 µm),
Span=(Dv90−Dv10)/Dv50, D₃₂, 점도제↔G'↔경도 다중스케일 연결.

**[논문 B] Ferrari et al. (2023)**, *CFD simulation of a high-shear mixer for food emulsion production*, J. Food Eng. 358:111655
로터-스테이터 고전단 믹서의 CFD-PBM 해석. 채택 개념: Taylor/토로이달 와류, 전단박화 겉보기 점도
ηr=ηr,∞+Kγ̇ᵐ, 모세관수 Ca와 임계 Ca, 혼합지수 α(전단 vs 신장), QMOM, D₃₂=M₃/M₂.

요약: 유동 시각화(토로이달)·점도 모델은 논문 B, PBM·파괴·스케일업 물리는 논문 A에서 도출.

## 2. 단계별 기록

각 항목: **프롬프트 요지** → 작업 결과 → 직전 한계 / 다음 동기.

### V1 — Canvas 2D 기본
프롬프트 요지: *Canvas 2D로 가마 그리고 입자 움직이게, 속도 슬라이더.*
결과: 가마 외형 + 랜덤 파티클 200개.
한계: 랜덤 이동이라 실제 교반 흐름 아님 → V2.

### V2 — 토로이달 순환 흐름 *(논문 B)*
프롬프트 요지: *랜덤 말고 중앙 하강·벽면 상승하는 토로이달 순환으로. 패들 RPM/온도 추가.*
근거: 논문 B의 로터-스테이터 Taylor 와류(토로이달 재순환)를 2D 속도장으로 단순화.
결과: 토로이달 속도장, 흐름 가이드.
한계: 호모 구조·흡입 없음 → V3.

### V3 — 호모·패들·스크래퍼 *(논문 B)*
프롬프트 요지: *하부 호모(로터) 추가, 흡입→분사 화살표, 패들·스크래퍼, 호모 RPM 0~3500.*
결과: 6엽 로터, 직선 패들, 스크래퍼, 흡입 표시, 유상→유화 전환.
한계: 패들 직선(비헬리컬), 깊이감 없음 → V4.

### V4 — 헬리컬 패들 3D
프롬프트 요지: *패들 헬리컬로, 입자 z좌표로 깊이감, 상별 색 구분, 유화율·입경·균일도 표시.*
결과: 헬리컬 패들, z-소팅 렌더, 물성 패널.
한계: 흡입구 단일, UI 미흡 → V5.

### V5 — 3패널 UI + 4단계 공정
프롬프트 요지: *좌측 컨트롤 + 우측 캔버스로 재구성, 4단계 공정 관리, 호모 내부 확대뷰.*
결과: 컨트롤/메인/호모확대 3분할.
한계: 두 뷰가 좁게 섞임 → V6.

### V6 — 탭 분리
프롬프트 요지: *탭으로 분리(유동역학 / 호모 확대), 배출구 슬롯 디테일.*
결과: 탭 UI, 슬롯 표현.
한계: Canvas 2D는 입체 회전·시점 변경 불가 → V7.

### V7 — Three.js WebGL 전환 ★대전환
프롬프트 요지: *Canvas 2D 버리고 Three.js로 재작성. importmap으로 OrbitControls 로드,
3D 메쉬, 설비 연동(설비코드/체적/충진율/토출량)·목표 점도/비중 패널.*
결과: Three.js 3D, 설비·운용 패널.
문제→해결: file:// importmap 차단 → 로컬 서버(`python -m http.server`) 안내.
한계: 물리값이 경험식 수준 → V9.

### V9.12 — 물리 모델 + 등-ε 스케일업 ★핵심 *(논문 A)*
프롬프트 요지: *N·L로 ε=b₁N³L² 계산, D₃₂ 산출(이론값 비교), 100/300/500L 등-ε 권장 RPM,
CSV·녹화.*
근거: 논문 A 식(7) ε=b₁N³L² 채택. 파괴가 ε·σ에 지배된다는 구조를 D₃₂에 반영. 스케일업은
"동일 ε → 동일 파괴 강도" 원리.
결과: Physics 모듈(ε, D₃₂), 등-ε 스케일업, CSV/녹화.
한계: 단일 D₃₂만, 분포·QC 없음 → V10.

### V10 — H-Index + 점도·경도 QC *(논문 A)*
프롬프트 요지: *Shannon Entropy 기반 균일도 H-Index(0~1) 추가, 점도(cps)·경도(mm)로 출하 QC 판정.*
근거: 논문 A의 DSD↔G'·경도 다중스케일 상관을 점도/경도·QC 근거로 사용. (H-Index는 논문에
없는 자체 고안 지표 — 분포 균일도를 정보엔트로피로 표현.)
한계: D₃₂ 단일 대표값 → V11.

### V11 — PBM 도입 ★핵심 *(논문 A)*
프롬프트 요지: *논문 A의 PBM 구현. breakage만(coalescence 무시), 파괴 빈도 관성+점성(K₁~K₄),
이항 파괴, Fixed-Pivot 65노드(0.01~40 µm), 히스토그램.*
근거: 논문 A는 점도제가 응집 억제 → 순수 파괴 가정. 파괴 빈도 식(4)(5), 수치해 식(8)(9) Fixed-Pivot.
문제→해결(v2): 시간 진행 시 개수/질량 보존 깨짐 → 논문 A가 강조한 Fixed-Pivot 재분배(ni,k)의
보존성을 반영해 가중치 보정, 안정화(`V11_PBM_v2`).
한계: 점도·경도 절대값이 실측과 동떨어짐 → V14.

### V14 — 점도·경도 재설계 *(논문 B)*
프롬프트 요지: *논문 B의 전단박화 겉보기 점도 ηr=ηr,∞+Kγ̇ᵐ를 골격으로, 화장품 cps 범위에
맞게 곱셈 보정. 분산상 부피분율·D₃₂가 점도/경도에 반영되게.*
근거: 논문 B 식(2) 채택. 농축 에멀젼에서 액적이 느끼는 점도가 겉보기 점도라는 가정 반영.
결과: 전단박화/곱셈 점도 모델, cps 범위 정합, QC 일관성 개선 (최종).

## 3. 회고

**잘 된 점** — 시각화·UI(V1~V7)를 빠르게 확보해 물리(V9~V14)에 집중. 논문 A·B 수식을 코드로
옮기는 출발점을 신속히 확보.

**직접 검증한 점** — (1) PBM 보존(V11→v2): 논문이 보존성을 보장한다 했으나 구현에서 깨져 재분배
가중치를 직접 디버깅. (2) 점도 절대값(V14): 논문 B는 마요네즈(고분산상)용이라 화장품 범위 보정이
필요. (3) 논문의 적용 조건(응집 무시, 점도제 1% 이상 유효)은 원문과 대조 확인.

**교훈** — AI는 논문 수식의 코드 초안을 빠르게 주지만, 적용 범위·수치 보존·단위 정합은 원문과
대조해 직접 검증해야 한다. 프롬프트에 논문명·수식·단위를 구체적으로 줄수록 결과가 좋았다.
